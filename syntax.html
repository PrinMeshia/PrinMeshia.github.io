Syntax Reference

Complete reference for the .webc language — all constructs, directives and built-ins.

app {\} layout {\} page {\} component {\} store {\} state {\} computed {\} props {\} @if / @else @for @for key= @error on:click= on:mount on:destroy emit() t() $store. $route. slot content link to= validate:

Application

The app block is the entry point. It declares the theme, layout and route table. 

app MyApp {
    theme: "default"
    layout: MainLayout
    routes {
        "/":      HomePage
        "/about": AboutPage
    \}
\}

Layout

Layouts wrap every page. Use slot content for the page injection point. Named slots support multi-zone layouts. 

// Single-slot layout
layout MainLayout {
    header { nav { link to="/" { "Home" \} \} \}
    main   { slot content \}
    footer { p "2025 My App" \}
\}

// Multi-zone layout (named slots)
layout DashLayout {
    header { slot header  \}
    aside  { slot sidebar \}
    main   { slot content \}
\}

Pages fill named slots with slot name { ... \} blocks. 

page "dashboard" {
    slot header  { h1 "Dashboard" \}
    slot sidebar { nav { link to="/" "Home" \} \}
    p "Main content"   // -> default content slot
\}

Elements and content

Any standard HTML tag is valid. Inline text or block children — both work. 

div class="container" {
    h1 "Hello world"
    p  "Plain text node"
    p  { "Mixed: " strong { "bold" \} " and plain." \}
    img src="/logo.png" alt="Logo"
    ul {
        li "Item one"
        li "Item two"
    \}
\}

Interpolation

Expressions inside {\} are evaluated reactively. They support arithmetic, comparisons, function calls and computed variables. 

p "Count: {count\}"
p "Next: {count + 1\}"
p "Max: {max(a, b)\}"
p "Full name: {fullName\}"    // computed var
div class={dynamicClass\} { \}

Control flow 

// Conditional
@if count > 0 {
    p "Positive"
\} @else {
    p "Zero or negative"
\}

// List — full re-render on change
@for item in items {
    li "{item\}"
\}

// List with key — DOM diffing (only changed nodes patched)
@for post key=post.id in posts {
    article "{post.title\}"
\}

// Validation error block
@error "email" {
    span class="error" "Invalid email"
\}
Key diffing: The key= expression must be unique within the list. Use item.id or item.slug . Added nodes are inserted, removed ones deleted, unchanged ones stay — preserving focus and animation state.

Events

Use on:<type>={expr\} on any element. Multiple instructions can be separated by ; . 

button on:click={count += 1\}                    { "+" \}
button on:click={count = max(0, count - 1)\}      { "−" \}
input  on:input={name = event.target.value\}
form   on:submit={handleSubmit\}

// Navigation
button on:click={webcore_navigate("/about")\} { "Go" \}

// Multi-instruction handler
button on:click={items = [...items, newItem]; draft = ""\} { "Add" \}

// Inter-component event
button on:click={emit("ping", count)\} { "Ping" \}

Attributes 

// Static string
div class="container" id="main"

// Boolean
input disabled
input required

// Dynamic expression
div class={dynamicClass\}
input value={count\}

// Validation attributes
input name="email"
      validate:required="Email is required"
      validate:email="Invalid address"
      validate:minlength="3,At least 3 chars"

Navigation 

// Declarative SPA link
link to="/about" { "About" \}
link to="/about" class="nav-link" { "About" \}

// Programmatic
button on:click={webcore_navigate("/about")\} { "Go" \}
button on:click={webcore_navigate("/user/" + userId)\} { "Profile" \}
Navigation uses history.pushState + fetch — no full page reload. on:destroy hooks run automatically before each navigation.

Parameterized routes v1.1.0

Route segments prefixed with : capture dynamic values. Access them in views with {$route.paramName\} . 

// src/app.webc
app MyApp {
    routes {
        "/":           HomePage
        "/post/:slug": PostPage
        "/user/:id":   UserPage
    \}
\}

// src/pages/post.webc
page "post" {
    h1 "Article: {$route.slug\}"
\}

// Navigate to a parameterized route
link to="/post/hello-world" { "Hello World" \}
button on:click={webcore_navigate("/user/" + userId)\} { "Profile" \}
The compiler generates a ROUTES[] array with RegExp patterns and a ROUTE_PARAMS object updated on every navigation. Both are tree-shaken when no parameterized routes are declared.
