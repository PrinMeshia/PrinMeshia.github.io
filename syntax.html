Syntax Reference

Complete reference for the .webc language syntax.

Application

The app block is the entry point. It declares the layout and route table. 

app MyApp {
    theme: "default"
    layout: MainLayout
    routes {
        "/":      HomePage
        "/about": AboutPage
    \}
\}

Layout

Layouts wrap every page. Use slot content for the page injection point. Named slots support multi-zone layouts. 

// Single-slot layout
layout MainLayout {
    header { nav { link to="/" { "Home" \} \} \}
    main   { slot content \}
    footer { p "2025 My App" \}
\}

// Multi-zone layout (named slots)
layout DashLayout {
    header { slot header  \}
    aside  { slot sidebar \}
    main   { slot content \}
\}

Pages fill named slots with slot name { ... \} blocks. 

page "dashboard" {
    slot header  { h1 "Dashboard" \}
    slot sidebar { nav { link to="/" "Home" \} \}
    p "Main content"   // -> default content slot
\}

Elements and content

Any standard HTML tag is valid. Inline text or block children — both work. 

div class="container" {
    h1 "Hello world"
    p  "Plain text node"
    p  { "Mixed: " strong { "bold" \} " and plain." \}
    img src="/logo.png" alt="Logo"
    ul {
        li "Item one"
        li "Item two"
    \}
\}

Interpolation

Expressions inside {\} are evaluated reactively at runtime. They support arithmetic, comparisons, function calls and computed variables. 

p "Count: {count\}"
p "Next: {count + 1\}"
p "Max: {max(a, b)\}"
p "Full name: {fullName\}"    // computed var
div class={dynamicClass\} { \}

Control flow 

// @if / @else -- reactive condition
@if count > 0 {
    p "Positive"
\} @else {
    p "Zero or negative"
\}

// @for -- reactive list render (full re-render on change)
@for item in items {
    li "{item\}"
\}

// @for with key -- key-based DOM diffing (v1.1.0)
// Only modified nodes are patched; focus and animation are preserved
@for post key=post.id in posts {
    article "{post.title\}"
\}

// @error -- validation error block
@error "email" {
    span class="error" "Invalid email"
\}

With key=expr the runtime compares keys before patching. Added nodes are inserted, removed nodes are deleted, unchanged ones stay untouched.

Events

Use on:<type>={expr\} on any element. The expression runs when the event fires. 

button on:click={count += 1\}                    { "+" \}
button on:click={count = max(0, count - 1)\}      { "-" \}
button on:click={count = 0\}                      { "Reset" \}
input  on:input={name = event.target.value\}
form   on:submit={handleSubmit\}

// Navigation
button on:click={webcore_navigate("/about")\} { "Go" \}

// Inter-component events
button on:click={emit("ping", count)\} { "Ping" \}

Attributes 

// Static string
div class="container" id="main"

// Boolean
input disabled
input required

// Dynamic expression
div class={dynamicClass\}
input value={count\}

// Validation
input name="email"
      validate:required="Email is required"
      validate:email="Invalid address"

Navigation 

// Declarative link (SPA-aware)
link to="/about" { "About" \}

// Programmatic
button on:click={webcore_navigate("/about")\} { "Go" \}
button on:click={webcore_navigate(root)\}      { "Home" \}

Navigation uses history.pushState + fetch — no full page reload. on:destroy hooks run automatically before each navigation.

Parameterized routes (v1.1.0)

Route segments prefixed with : capture dynamic values. Access them in views with . 

// src/app.webc
app MyApp {
    routes {
        "/":           HomePage
        "/post/:slug": PostPage
        "/user/:id":   UserPage
    \}
\}

// src/pages/post.webc
page "post" {
    h1 "Article: {$route.slug\}"
    p  "Reading post ID: {$route.slug\}"
\}

// Navigation to a parameterized route
link to="/post/hello-world" { "Hello World" \}
button on:click={webcore_navigate("/user/" + userId)\} { "Profile" \}

The compiler generates a ROUTES[] array with RegExp patterns. ROUTE_PARAMS is updated on every navigation and at page load. Both arrays are tree-shaken when no parameterized routes are declared.
