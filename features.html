Global store

The store { \} block declares reactive state shared across all components and pages. 

// app.webc
store {
    count:  Number  = 0
    theme:  String  = "dark"
    active: Boolean = false
\}

// Any component — prefix with $store.
component Display {
    view {
        p "Global count: {$store.count\}"
        button on:click={$store.count += 1\} { "+" \}
    \}
\}

component Badge {
    view {
        @if $store.active {
            span "Active"
        \} @else {
            span "Inactive"
        \}
    \}
\}
Scoped reactivity: Store vars use the same S.set() / S.get() machinery as component state. Any binding that reads $store.x automatically re-renders when $store.x changes — no subscription boilerplate.

Form validation

Declarative validation attributes on inputs — no JS event wiring needed. 

component SignupForm {
    view {
        form on:submit={handleSubmit\} {
            label "Username"
            input type="text" name="username"
                  validate:required="Username is required"
                  validate:minlength="3,At least 3 characters"
            @error "username" {
                span class="error-msg" "Check your username"
            \}
            label "Email"
            input type="email" name="email"
                  validate:required="Email is required"
                  validate:email="Must be a valid email"
            @error "email" {
                span class="error-msg" "Check your email"
            \}
            button type="submit" { "Sign up" \}
        \}
    \}
\}

Validation fires on blur and submit . @error blocks are hidden by default; shown when the corresponding field fails.

Attribute Format Description
validate:required "message" Field must not be empty
validate:minlength "n,message" Minimum character count
validate:maxlength "n,message" Maximum character count
validate:email "message" Email format check
validate:pattern "regex,message" Custom regex
Validation logic uses the browser's native input.checkValidity() capture phase, so no custom regex engine is shipped. The validateField() helper is tree-shaken out if no validate:* attributes appear in the document.

Static Site Generation (SSG)

The compiler pre-renders initial state into HTML at build time. No flash of unstyled or wrong content on first paint. 

// .webc source — state: count = 0
p "Counter: {count\}"
@if count > 0 { p "Positive" \} @else { p "Zero" \}

// HTML output (SSG pre-rendered)
<p>Counter: <span data-webcore-interpolation="count">0</span></p>
<div data-webcore-if="count > 0" style="display:none">...</div>
<div data-webcore-else="count > 0" style="display:block">...</div>
Zero JS on first paint: The server sends fully populated HTML. The runtime attaches to existing DOM nodes via data-webcore-* attributes rather than replacing them, so the initial render is instant and crawlable.

Internationalisation (i18n)

Place .toml files in locales/ . Use t("key") in views. Switch locale at runtime with setLocale(code) . 

// locales/en.toml
welcome   = "Welcome"
increment = "Increment"

// locales/fr.toml
welcome   = "Bienvenue"
increment = "Incrementer"

// Component view
h1 "{t("welcome")\}"
button on:click={count += 1\} { "{t("increment")\}" \}
button on:click={setLocale("fr")\} { "FR" \}
button on:click={setLocale("en")\} { "EN" \}

i18n: parameters and pluralization v1.1.0

Pass a second argument to t() for pluralization or positional substitution. 

// locales/en.toml
items_one   = "

} item"
items_other = "

} items"
greeting    = "Hello, 

}!"

// locales/fr.toml
items_one   = "

} element"
items_other = "

} elements"
greeting    = "Bonjour, 

} !"

// Pluralization — picks _one or _other key
p "{t("items", 1)\}"       // → "1 item"
p "{t("items", count)\}"   // → "3 items" (reactive)

// Positional substitution
p "{t("greeting", username)\}"  // → "Hello, Alice!"
Call Behaviour
t("key") Simple lookup — returns the value or the key if missing
t("key", n) Number → picks _one (n=1) or _other, replaces }
t("key", val) Non-number → replaces } with val

WebAssembly (WASM)

Place a Rust crate in wasm/ . The compiler detects it, runs wasm-pack , and injects an async loader — no manual wiring. 

// wasm/src/lib.rs
use wasm_bindgen::prelude::*;
#[wasm_bindgen]
pub fn compute(n: u32) -> u32 { n * n \}

// WebCore component
component WasmDemo {
    state { result: Number = 0 \}
    view {
        button on:click={result = wasm.compute(result)\} { "Compute" \}
        p "Result: {result\}"
    \}
\}
globalThis.wasm is populated asynchronously after the module loads. A full bind() call fires on success so any interpolation using wasm.* reflects the initial state immediately. The WASM loader is tree-shaken out when no wasm/ directory is present.

Runtime tree-shaking

The compiler emits only the runtime functions actually used by the document. Unused helpers are never written to the output bundle. 

// Minimal bundle — no @for, @if, validation, routing, or WASM
class State {
  #d=new Map(); #l=new Map();
  set(k,v){ this.#d.set(k,v); this.#l.get(k)?.forEach(f=>f(v)) \}
  setQ(k,v){ this.#d.set(k,v) \}
  get(k){ return this.#d.get(k) \}
  on(k,f){ (this.#l.get(k)??this.#l.set(k,[]).get(k)).push(f) \}
\}
const S = new State();
// bindIf, bindFor, bindAttrs, nav, validateField — absent
Function / constant Emitted when…
bindFor() At least one @for loop
bindIf() At least one @if / @else
bindAttrs() At least one attr={expr\} dynamic attribute
validateField() At least one validate:* or @error
nav() + popstate Routes declared or webcore_navigate() used
DESTROY_HOOKS At least one on:destroy {\} block
COMPUTED + rebindComputed() At least one computed {\} block
WASM loader wasm/Cargo.toml present at build time
How it works: detect_features() in codegen_js.rs walks the AST once before code generation and sets boolean flags per feature. Each emitter checks its flag before writing — unused helpers are never serialised to the output file.
