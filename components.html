Component Model

Props · State · Computed · Lifecycle · Inter-component events · Scoped CSS

Anatomy of a component

Every block is optional except view — the HTML tree is required. 

component MyComponent {
    props    { ... \}      // pass-in values (static or reactive)
    state    { ... \}      // local reactive state
    computed { ... \}      // derived values, re-evaluated before each bind
    on:mount   { ... \}    // runs at DOMContentLoaded
    on:destroy { ... \}    // runs before SPA navigation / window.beforeunload
    view     { ... \}      // required — the HTML tree
    style    { ... \}      // scoped CSS (FNV-1a data-v hash)
\}

Props

Props are values passed at call site. Static props ( name="value" ) are substituted at compile time. Dynamic props ( name={expr\} ) compile to reactive spans. 

component Badge {
    props { label: String, color: String \}
    view  { span class={color\} "Status: {label\}" \}
\}

// Static props — substituted at compile time
Badge label="Active" color="green" {\}

// Dynamic props — reactive at runtime
Badge label={statusMsg\} color={statusColor\} {\}

Compound expressions using a prop name also work: 

component StepCounter {
    props { step: Number \}
    view {
        p "Next: {step + 1\}"        // compound expression
        span class={color\} "Status" // prop as dynamic attribute
    \}
\}
StepCounter step="2" color="green" {\}

State

Local reactive state. Any assignment triggers a DOM update for bindings that depend on it. 

state {
    count:  Number  = 0
    name:   String  = "World"
    active: Boolean = true
    items:  List    = []
\}
Types: Number · String · Boolean · List . The type annotation determines the SSG pre-render strategy.

Computed — derived state

The computed block declares variables derived from state. Re-evaluated before every DOM bind. 

component FullName {
    state {
        firstName: String = "Jean"
        lastName:  String = "Dupont"
    \}
    computed {
        fullName = firstName + " " + lastName
        initials = firstName[0] + "." + lastName[0]
    \}
    on:mount { firstName = "Marie" \}
    view { p "Hello {fullName\} ({initials\})" \}
\}
No reactive loops: Computed vars use S.setQ() (silent setter) — state is updated without triggering listeners. Safe to derive from the same vars they depend on, with zero risk of infinite cycles.

Lifecycle — on:mount

on:mount { \} runs once at DOMContentLoaded , after the reactive runtime is ready. Wrapped in an IIFE for variable isolation. 

component Clock {
    state { time: String = "" \}
    on:mount {
        const tick = () => {
            S.set("time", new Date().toLocaleTimeString());
            bind();
        \};
        tick();
        window.__clockId = setInterval(tick, 1000);
    \}
    on:destroy { clearInterval(window.__clockId); \}
    view { p "Time: {time\}" \}
\}

Lifecycle — on:destroy

on:destroy { \} runs before every SPA navigation and on window.beforeunload . Use it to clean up timers, cancel requests, or deregister listeners. 

// Compiled output (tree-shaken in only when used)
const DESTROY_HOOKS = [
    () => { clearInterval(window.__clockId); \}
];
function runDestroyHooks() { DESTROY_HOOKS.forEach(h => h()); \}
window.addEventListener('beforeunload', runDestroyHooks);
DESTROY_HOOKS and runDestroyHooks are emitted only when at least one on:destroy block is present in the document — full tree-shaking.

Inter-component events

emit("name", data) dispatches a CustomEvent on document. Listen with on:name={handler\} at the call site. 

// Emitter component
component Notifier {
    view {
        button on:click={emit("ping", count)\} { "Ping" \}
        button on:click={emit("reset")\}       { "Reset" \}
    \}
\}

// Call site — subscribe to events
page "home" {
    Notifier on:ping={total += 1\} on:reset={total = 0\} {\}
    p "Received {total\} pings"
\}
Events are dispatched on document — they are global. The payload is accessible via e.detail in the handler expression.

Scoped style and @media

Every component gets a unique data-v attribute computed as a deterministic FNV-1a hash of the component name. @media queries inside style { \} are scoped automatically. 

component Card {
    view { div class="card" { slot content \} \}
    style {
        .card { padding: 1rem; background: #1e293b; \}
        @media (max-width: 768px) {
            .card { padding: 0.5rem; \}
        \}
    \}
\}
// Output: .card[data-v="a1b2"] { ... \}
//         @media (...) { .card[data-v="a1b2"] { ... \} \}
