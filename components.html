Component Model

Props · State · Computed · Lifecycle · Events · Routes

Anatomy of a component 

component MyComponent {
    props    { ... \}      // pass-in values
    state    { ... \}      // local reactive state
    computed { ... \}      // derived values
    on:mount   { ... \}    // runs at DOMContentLoaded
    on:destroy { ... \}    // runs before navigation / unload
    view     { ... \}      // required -- the HTML tree
    style    { ... \}      // scoped CSS
\}

Props

Props are values passed into a component at call site. They accept static strings or dynamic expressions. 

component Badge {
    props { label: String, color: String \}
    view  { span class={color\} "Status: {label\}" \}
\}

// Static props
Badge label="Active" color="green" {\}

// Dynamic props -- reactive
Badge label={statusMsg\} color={statusColor\} {\}

A static prop ( name="value" ) is substituted at compile time. A dynamic prop ( name={expr\} ) compiles to a reactive interpolation span.

Compound expressions using a prop name also work (v1.1.0): 

component StepCounter {
    props { step: Number \}
    view {
        // {step + 1\} — prop used inside a compound expression
        p "Next step: {step + 1\}"
        // class={color\} — prop used as a dynamic attribute
        span class={color\} "Status"
    \}
\}

// Call site
StepCounter step="2" color="green" {\}
// Renders: "Next step: (2) + 1" (evaluated at runtime)
//          <span class="green">

State

Local reactive state. Any assignment triggers a DOM update. 

state {
    count:  Number  = 0
    name:   String  = "World"
    active: Boolean = true
    items:  List    = []
\}

Types: Number · String · Boolean · List

Computed — derived state

The computed block declares variables derived from state. Re-evaluated before every DOM bind. 

component FullName {
    state {
        firstName: String = "Jean"
        lastName:  String = "Dupont"
    \}
    computed {
        fullName = firstName + " " + lastName
        initials = firstName + "." + lastName
    \}
    on:mount { firstName = "Marie" \}
    view { p "Hello {fullName\} ({initials\})" \}
\}

Computed vars use S.setQ() (silent setter) — no listener triggers, no reactive loops.

Lifecycle — on:mount

on:mount { \} runs once in DOMContentLoaded , after the runtime is ready. Wrapped in an IIFE for variable isolation. 

component Clock {
    state { time: String = "" \}
    on:mount {
        const tick = () => { S.set("time", new Date().toLocaleTimeString()); bind(); \};
        tick();
        window.__clockId = setInterval(tick, 1000);
    \}
    on:destroy { clearInterval(window.__clockId); \}
    view { p "Time: {time\}" \}
\}

Lifecycle — on:destroy

on:destroy { \} runs before every SPA navigation and on window.beforeunload . Use it to clean up timers, cancel requests, or deregister listeners. 

// Runtime output (tree-shaken in)
const DESTROY_HOOKS = [
    () => { clearInterval(window.__clockId); \}
];
function runDestroyHooks() { DESTROY_HOOKS.forEach(h => h()); \}
window.addEventListener('beforeunload', runDestroyHooks);

DESTROY_HOOKS and runDestroyHooks are emitted only when at least one on:destroy block is present.

Inter-component events

emit("name", data) dispatches a CustomEvent on document. Listen with on:name={handler\} at the call site. 

// Emitter component
component Notifier {
    view {
        button on:click={emit("ping", count)\} { "Ping" \}
        button on:click={emit("reset")\}       { "Reset" \}
    \}
\}

// Listening to events at call site
page "home" {
    Notifier on:ping={total += 1\} on:reset={total = 0\} {\}
    p "Received {total\} pings"
\}

Events are global (document). Payload accessible via e.detail in handler.

Scoped style and @media

Every component gets a unique data-v attribute (FNV-1a hash). Media queries inside style { \} are scoped automatically. 

component Card {
    view { div class="card" { slot content \} \}
    style {
        .card { padding: 1rem; background: #1e293b; \}
        @media (max-width: 768px) {
            .card { padding: 0.5rem; \}
        \}
    \}
\}
// Output: .card[data-v="a1b2"] { ... \}
//         @media (...) { .card[data-v="a1b2"] { ... \} \}
