Examples

Full projects built with WebCore — run any of them locally with webc dev

Included examples

All examples live under examples/ in the repository. Each compiles to a standalone dist/ — no framework, no bundler, no client-side dependencies.

counter

Counter

Minimal reactive counter with increment and decrement. The simplest possible WebCore app — ideal starting point. 

cd examples/counter
webc dev
todo

Todo

Todo list with add, toggle done, and delete. Demonstrates @for, @if, and reactive state with computed values and @for key diffing. 

cd examples/todo
webc dev
blog

Blog

Multi-page blog with parameterized routes (/post/:slug). Shows SPA routing, $route params, and global store for theme switching. 

cd examples/blog
webc dev
forms

Forms

SignupForm and ContactForm with full declarative validation — required, email, minlength, maxlength, pattern, and @error blocks. 

cd examples/forms
webc dev
i18n

i18n

Internationalisation demo — t() calls, plural forms (items_one / items_other), positional params, and reactive setLocale(). 

cd examples/i18n
webc dev
docs

Docs

This documentation site — 5 pages, SPA routing, named slots, copy-to-clipboard, active nav, and a minimal design system. 

cd examples/docs
webc dev

Project structure

Every example follows the same layout: 

my-project/
├── webc.toml          # project name, entry point
├── theme.toml         # design tokens (colors, spacing, fonts)
├── public/
│   └── styles.css     # optional extra styles
└── src/
    ├── app.webc       # routes + layout declaration
    ├── layouts/
    │   └── Main.webc
    ├── pages/
    │   └── home.webc
    └── components/
        └── Counter.webc

Scaffold a new project

Use webc new to generate a full project skeleton — webc.toml, theme.toml, layout, homepage, and a Counter component. 

webc new my-app
cd my-app
webc dev

The scaffold generates a working Counter component with reactive state and scoped CSS, so you have something to build on immediately.

Building for production

Run webc build to produce a minified, production-ready dist/ — CSS is processed with LightningCSS, JS is stripped of comments and whitespace. 

webc build        # minified output in dist/
webc build dev    # un-minified, easier to inspect
