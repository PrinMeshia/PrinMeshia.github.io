Build web interfaces declaratively

WebCore unifies HTML, CSS and JavaScript into a single .webc syntax. The Rust compiler generates semantic HTML, scoped CSS and a minimal reactive runtime — no framework, no bundler.

Read the Docs Component Model 
component Counter {
    state  { count: Number = 0 \}
    computed { doubled = count * 2 \}
    view {
        div {
            p "Value: {count\} * 2 = {doubled\}"
            button on:click={count += 1\} { "+" \}
        \}
    \}
    style { div { display: flex; gap: 1rem; \} \}
\}

Why WebCore?

⚡ Performance

Zero runtime dependencies

The compiler emits only the JS helpers your document actually uses. Simple apps get a tiny bundle.

🦀 Reliability

Rust-powered compiler

A Pest PEG grammar drives a fully typed AST. Errors include file, line, column and source context.

🔒 Isolation

Scoped CSS by default

Each component gets a deterministic FNV-1a hash. Styles never leak across component boundaries.

🌐 Routing

SPA routing built-in

History API navigation with no full page reload. Routes are declared directly in the app block.

🎛️ Reactivity

Reactive state

ES2022 private-field State class. Computed vars, lifecycle hooks, and inter-component events included.

📦 SSG

Static Site Generation

Initial state values are pre-rendered at compile time. No flash of unstyled or wrong content.

Get started in seconds

1
Clone and build 
git clone https://github.com/PrinMeshia/Webcore.git
cd Webcore/webcore-compiler
cargo build --release
2
Scaffold a new project 
./target/release/webcore-compiler new my-app
3
Start the dev server 
cd my-app && webc dev 3000
