Build web interfaces declaratively

WebCore compiles .webc files to semantic HTML, scoped CSS and a minimal reactive runtime. One language, one compiler, zero dependencies.

component Counter {
    state   { count: Number = 0 \}
    computed { doubled = count * 2 \}
    view {
        div {
            p "Value: {count\} — doubled: {doubled\}"
            button on:click={count += 1\} { "+" \}
            button on:click={count = max(0, count-1)\} { "−" \}
        \}
    \}
    style { div { display: flex; gap: 1rem; \} \}
\}

Why WebCore?

One declarative syntax that replaces HTML templates, CSS modules, and JS reactivity boilerplate.

⚡

Tree-shaken runtime

Performance

The compiler emits only the JS helpers your document actually uses. Simple apps ship under 5 KB of script.

🦀

Rust + PEG parser

Reliability

A Pest grammar drives a fully-typed AST. Build errors include file, line, column and a source caret.

🔒

Scoped CSS

Isolation

Each component gets a deterministic FNV-1a hash. Styles never leak. No BEM, no CSS modules.

🌐

SPA routing

Routing

History API navigation with parameterised routes (/post/:slug). Declare routes in the app block.

🎛️

Reactive state

Reactivity

ES2022 private-field State. Computed vars, lifecycle hooks, inter-component events, global store.

📦

SSG + i18n + WASM

Features

Pre-render initial values, ship translated strings from TOML, and call Rust WASM from any component.

Get started

1
Clone and build the compiler 
git clone https://github.com/PrinMeshia/Webcore.git
cd Webcore/webcore-compiler && cargo build --release
2
Scaffold a new project 
./target/release/webcore-compiler new my-app
3
Start the dev server 
cd my-app && webc dev 3000
